1. 读取输入

1.1 Scanner

使用 Scanner 对象中的 next() 、nextLine() 方法可以从控制台读取字符串。

形如：

Scanner in = new Scanner(System.in);
System.out.print("What's your name? ");
String name = in.nextLine();

Scanner 对象中没有读取单个字符的方法，因而需要在读取字符串后，提取对应的码点/字符。

类似的，我们可以使用 Scanner 类的 nextByte() 、nextShort() 、nextInt() 、nextLong() 、nextFloat() 、nextDouble() 方法来获取相应类型的数据。

需要注意的是，为了避免出现错误，我们不应在上述方法（包括 next() 方法）之后使用 nextLine() 方法。

因为这些方法都是 标记读取方法（token-reading method） ，会读取直到分隔符前的输入，但不读取分隔符本身（分隔符一般是空格、回车等，nextLine() 方法以回车符为分隔符），因而可能出现如下问题：

Scanner input=new Scanner(System.in);

int n=input.nextInt();

String line=input.nextLine();

对于上面这段代码，我们输入 7 并回车，再输入 17 并回车，结果将是 n 为 7， line 是一个空的字符串。

原因很简单，nextInt() 方法读取了我们输入的 7 之后，在分隔符（在上面的输入里，回车符是分隔符）前停止了而 没有读取这个分隔符本身，导致这个分隔符仍存在于缓冲区里，并成为了缓冲区内的第一个值。于是 nextLine()方法 会从这个分隔符开始读取输入，并在一个行分隔符前结束。正巧的是，这个分隔符就是行分隔符，于是这个 nextLine() 方法将即刻停止下来，什么输入都不能读取到，返回给 line 的字符串便成了一个空的字符串。

1.2 Console

当然，由于 Scanner 类的输入是可见的，它不适用于从控制台读取密码。Java 6 特别引入了 Console 类来实现这个目的，如：

Console cons = System.console();
String username = cons.readLine("User name: ");
char[] password = cons.readPassword("Password: ");

为安全起见，返回的密码存放在一个 char 数组中，而不是字符串中。在对密码完成处理之后，应该马上用一个填充值覆盖数组元素。

Console 类没有能够读取单个词或值的方法。

2. 输出

直接在命令行中输出内容，有如下三类：

// 输出
System.out.print("Hello World!");

// 输出并换行
System.out.println("Hello World!");

// 格式化输出
System.out.printf("Hello World!");

对于新的代码，我们应该使用 java.time 包中的方法作为 Date 类的代替，不过你可能会在遗留代码中看到 Date 类和相关的格式化选项（%tx 或 %Tx ，x 的具体类型见下表）：

从表中可以看到，某些格式只给出了指定日期的部分信息，例如，只有日期或月份。如果需要多次提供日期来分别格式化每一部分，就太麻烦了。为此可以用一个格式字符串，即 参数索引（argument index） 来指示要格式化的参数 索引（索引值从 1 开始，避免与 0 标志混淆） 。

索引必须紧跟在 % 后面，并以 $ 终止：

System.out.printf("%1$s %2$tB %2$te, %2$tY", "Due date: ", new Date()); // -> Due date: Fabruary 9, 2015

还可以选择使用 < 标志。它只是前面格式说明中的参数将被再次使用，如：

System.out.printf("%s %tB %<te, %<tY", "Due date: ", new Date()); // -> Due date: Fabruary 9, 2015

3. 文件输入输出

这里只是简单介绍了基本的文件输入输出方法，详细请见 Java IO 流 。

3.1 读取文件

读取文件，形如：

File file = new File("myfile.txt");
Scanner in = new Scanner(file, StandardCharsets.UTF_8);

在这里，我们指定了 UTF-8 字符编码。

注意，如果构造的是一个带有字符串参数的 Scanner ，它会 把字符串解释为数据，而不是文件名 ，如：

Scanner in = new Scanner("myfile.txt");

上述代码会将参数看为包含 10 个字符（'m' 、'y' 等）的数据。

3.2 写入文件

写入文件，形如：

PrintWriter out = new PrintWriter("myfile.txt", StandardCharsets.UTF_8);

如果文件不存在，PrintWriter 将创建这个文件。我们可以像输出到 System.out 一样使用 print 、println 、printf 。

3.3 注

当指定一个相对文件名是，例如 "myfile.txt" 、"mydirectory/myfile.txt" 或 "../myfile.txt" ，文件位于相对于 Java 虚拟机启动目录的位置。

如果在命令行方式下执行 java MyProgram 命令启动程序，启动目录就是命令解释器的当前目录。

使用 IDE 时，启动目录将由 IDE 控制。

可以使用如下方法得到启动目录的位置：

String dir = System.getProperty("user.dir");

如果觉得定位太麻烦，也可以使用绝对路径。

需要注意的是，如果使用一个不存在的文件构造一个 Scanner ，或者用一个无法创建的文件名构造一个 PrintWriter ，就会产生 IOException 异常。

当采用命令行方式启动一个程序，可以利用 Shell 重定向语法，将任意文件关联到 System.in 和 System.out ，这样就不必担心处理 IOException 异常了：

java MyProg < myfile.txt > output.txt
