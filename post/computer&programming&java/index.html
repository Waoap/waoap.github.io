1. 计算机

计算机是存储和处理电子数据的电子设备。

计算机包括 硬件（Hardware） 和 软件（Software） 两部分。一般来说，硬件指计算机中可以看得见的物理部分，而软件提供不可见的指令，这些指令控制硬件并使硬件完成特定的任务。

2. 硬件

一台计算机由以下几个主要的硬件组件构成：

它们通过一个称为 总线（Bus） 的子系统连接。在个人电脑计算机中，总线内置在主板中，主板是一个将计算机各个部分连接在一起的电路板。

2.1 中央处理器

中央处理器（Central Processing Unit，CPU） 是计算机的大脑，它从内存中获取指令并执行这些指令。

CPU 通常由 控制单元（Control Unit） 和 算数/逻辑单元（Arithmetic/Logic Unit） 两部分组成。

  • 控制单元用于控制和协调其他组件的动作

  • 算术/逻辑单元用于完成 数值运算（加法、减法、乘法、除法）和逻辑运算（比较）

现在的 CPU 构建在一块小小的硅半导体芯片上，其上包含数百万称为晶体管的小电路开关，用于处理信息。

每台计算机都有一个 内部时钟 ，以固恒定速率发射电子脉冲，用于控制和同步各种操作的步调，其发射频率也即是我们常说的 CPU 频率，计量单位为 赫兹 （Hertz，Hz） 。如今较新的桌面级 CPU ，其时钟频率一般为 3 ~ 5 GHz。

最初开发的 CPU 只有一个 核（Core） 。核是 CPU 中实现指令读取和执行的部分。现在的消费级 CPU 一般拥有 2 ~ 8 单独核心。

2.2 内存

2.2.1 比特（bit）和字节（byte）

在讨论内存之前，我们需要知道信息（数据和程序）是如何存储在计算机中的。

实际上，计算机不过是一系列的电路开关，开、关两种状态分别对应数值 1 、0 。这些 0 和 1 被解释为为二进制系统中的数，称为 比特（Bit，二进制数）。

在计算机中，比特不是最小的存储单位，字节（Byte） 才是。其中 1 Byte = 8 Bit 。

在计算机中，各种类型的数据（例如，数字和字符）都被编码为字节的序列，其编码和解码都是系统根据 编码方案（Scheme） 来自动完成的。

2.2.2 内存

计算机的内存由一个 有序的字节序列 组成，用于存储程序及程序正使用的数据。程序及其数据必需移至计算机的内存，然后才能由 CPU 执行。

内存中的每个字节都有 唯一的地址 。由于可以按任意顺序存取字节，内存也被称为 随机访问存储器（Random-Access Memory，RAM） 。

内存是一种易失的数据存储形式——断电时存储在内存中的信息会丢失。

当今的个人计算机通常有 8 ~ 32 GB 的内存。通常而言，计算机具有的内存越多，其运行速度越快，但这条经验法则是有局限性的。

内存字节永远不会为空 ，但其初始内容对于我们的程序可能毫无意义。一旦新的信息被放入，该内存字节先前的内容将会丢失。

和 CPU 一样，内存也是构建在一个表面嵌有数百万个晶体管的硅半导体芯片上。与 CPU 芯片相比，内存芯片更简单、更低速，也更便宜。

2.3 存储设备

程序和数据被永久地存放在存储设备上，并在计算机实际使用它们时再移入内存，在内存中的操作速度比在永久存储设备中快得多。

存储设备主要有以下四种类型：

  1. 磁盘驱动器

  2. 光盘驱动器（CD 和 DVD）

  3. 通用串行总线（USB）闪存驱动器

  4. 云存储

驱动器（Drive） 是对存储介质（例如，磁盘和光盘）进行操作的设备。存储介质物理地存储数据和程序指令。驱动器从介质读取数据，以及将数据写入介质。

上述驱动器分别对应如下介质：

  1. 磁盘：

    包括硬盘（Hard Disk）和移动硬盘（Mobile Hard Disk）。

  2. 光盘和数字化视频磁盘（CD 和 DVD）：

    CD 分为只读光盘（CD-ROM）、可录光盘（CD-R）和可复写光盘（CD-RW），DVD 分为可录 DVD（DVD-R）、可复写 DVD（DVD-RW）。

  3. USB 闪存：

    通用串行总线（Universal Serial Bus，USB）接口允许用户将多种外部设备连接到计算机，包括 USB 闪存驱动器（flash drive）（即我们常说的 u 盘）。

  4. 云存储：

    如各种网盘等云存储平台。

2.4 输入、输出设备

  • 常用的输入设备：

    包括键盘、鼠标、麦克风、摄像头等。

  • 常用的输出设备：

    包括显示器、音响、打印机等。

2.5 通信设备

如拨号调制解调器（Modulator/Demodulator，调制器/解调器）、数字用户线（Digital Subscribe Line，DSL）、电缆调制解调器、有线网卡接口、无线适配器等。

3. 软件（程序）

计算机程序被称为软件，是告诉计算机做什么的指令。

3.1 操作系统

操作系统（Operating System, OS) 是运行在计算机上的最重要的程序。操作系统可以管理和控制计算机的活动。

操作系统的主要任务有：

  1. 控制和监视系统活动

    识别输入、发送输出结果、跟踪存储设备中的文件和文件夹、控制外部设备、确保不同程序和用户同时使用计算机时不会相互干扰、对安全性负责等。

  2. 分配和调配系统资源

    确定一个程序需要使用哪些计算机资源（例如，CPU、内存、磁盘、输入和输出设备），并进行资源分配和调配以运行程序。

  3. 调度操作

    调度程序活动，以便有效地利用系统资源。

    为了提高系统的性能，目前许多操作系统都支持如多道程序设计（Multiprogramming）、多线程（Multithreading）和多处理（Multiprocessing）这样的技术。

    • 多道程序设计允许多个程序通过共享同一个 CPU 同时运行

    • 多线程允许单个程序同时执行多个任务

    • 多处理类似于多线程，区别在于其是采用多个处理器来并行允许多个程序

4. 设计软件的工具——编程语言

计算机程序（Program） 必须使用计算机可以使用的语言编写，编程语言正是用来设计计算机程序的。

编程语言包括 机器语言 、汇编语言（亦称低级语言） 、高级语言 三类，Java 便属于高级语言。

4.1 机器语言

计算机的原生语言就是 机器语言（Machine Language） ，即一套内嵌的原子指令集，其因计算机类型的不同而有差异。这些指令都以二进制的形式存在，因而编写程序时必须以二进制代码输入指令。

4.2 汇编语言（亦称低级语言）

由于使用机器语言进行程序设计过于复杂，因而在计算的早期就创建了汇编语言作为替代。

汇编语言（Assembly Language） 使用简短的描述性单词（称为助记符）来表示每一条机器语言指令，其出现降低了程序设计的难度。然而，由于计算机不理解汇编语言，所以需要使用一种称为 汇编器（Assembler） 的程序将汇编语言程序转换为机器代码。

汇编语言被认为是低级语言，其语言中的指令本质上与机器代码中的指令相对应。汇编语言本质上非常接近机器语言，并且是依赖机器的。

4.3 高级语言

20 世纪 50 年代，新一代编程语言，即众所周知的 高级语言 出现了。它们是平台独立的，这意味着可以使用高级语言编程，然后在各种不同类型的机器上运行。

高级语言的指令称为语句，其编写的程序称为 源程序（Source Program） 或 源代码（Source Code） 。由于计算机不能运行源程序，必须被转换成可执行的机器代码。其中，这种转换可以由一种称为 解释器（Interpreter） 或 编译器（Compiler） 的编程工具来完成。

  1. 解释器的工作原理：

    解释器从源代码中读取一条语句，将其转换为机器代码或者虚拟机器代码，然后立刻运行。来自源代码的一条语句可能被转换为多条机器指令。

  2. 编译器的工作原理：

    编译器将整个源代码转换为机器代码文件，然后执行该机器代码文件。

5. 高级语言之一——Java

Java 最早由 Sun 公司发布于 1996 年，2010 年 Sun 公司为 Oracle 公司收购。如今我们使用的一般是 Oracle Java。

Java 最核心的特性是 跨平台与可移植性 和 垃圾回收机制 。

5.1 特性

  1. 简单的（Simple）

  2. 面向对象的（Object-Oriented）

  3. 分布式的（Distributed）

  4. 解释型的（Interpreted）

  5. 健壮的（Robust）

  6. 安全的（Secure）

  7. 架构中立的（Architecture-Neutral）

  8. 可移植的（Portable）

  9. 高性能的（High-Performance）

  10. 多线程的（Multithreaded）

  11. 动态的（Dynamic）

5.2 核心特性

5.2.1 跨平台与可移植性

可移植性，指同一个程序可以在不同操作系统之间任意地进行部署。

Java 中，编译器 将源文件 编译 为字节码文件，并由适合各个平台的 JVM（Java Virtual Machine） 将字节码文件 解释 成机器码并 执行 。其中，正是 JVM 充当着翻译官的角色，将字节码文件解释成各平台对应的可执行文件，从而实现跨平台与可移植性。

注：因此我们也认为，Java 既是一种 “编译型语言” ，也是一种 “解释型语言”。

5.2.2 垃圾回收机制

垃圾（内存）回收的目的在于主动清除不再使用的对象等，释放内存，减少了因程序设计时没有释放空间而造成内存泄漏的情况的发生。

5.3 不同版本（Edition）

Java 分为如下三个版本（Edition）：

  1. Java Standard Edition，Java SE：

    标准版，用于开发客户端的应用程序

  2. Java Enterprise Edition，Java EE：

    企业版，用于开发服务端的应用程序，如 Java servlet 、JavaServer Pages（JSP）以及 JavaServer Faces（JSF）

  3. Java Micro Edition，Java ME：

    微型版，用于开发移动设备上的应用程序

5.4 一些术语

Java 语言规范（Java Language Specification） 是对语言的技术定义，包括 Java 程序设计语言的语法和语义。完整的 Java 语言规范可以在 https://docs.oracle.com/javase/specs/ 上找到。

应用程序接口（Application Program Interface，API） 也称为库，包含了为开发 Java 程序而预定义的类和接口。API 仍然在扩展，在 Oracle 官网上，可以査看和下载对应版本的 Java API。

Java 开发工具包（Java Development Toolkit，JDK） 是由一套独立程序构成的集合，用于开发和测试 Java 程序。

Java 运行环境（Java Runtime Environment，JRE） ，用于运行 Java 程序。

集成开发环境（Integrated Development Environment，IDE） ，用于快速开发 Java 程序。

软件工具开发包（Software Development Kit，SDK），过时的术语，用于描述旧版本的 Java JDK。

6. 本文的重要术语

  1. 中央处理器（Central Processing Unit，CPU）：

    计算机的大脑，它从内存中获取指令并执行这些指令。

  2. 比特（Bit）：

    二进制数值的长度单位，指二进制数的长度（大小）为一位，如 1 或 0 的长度（大小）是 1 bit

  3. 字节（Byte）：

    二进制数值的长度单位，1 Byte = 8 Bit ，如 01010101 的长度（大小）为 1 Byte
