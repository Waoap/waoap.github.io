1. 类

1.1 类的概念和定义形式

在 Java 程序中最基础的单元是 类（Class），所有的程序都必须封装在类中执行。

类的定义形式如下：

class <CLASS_NAME> {
    . . .
}

1.2 外部类、内部类和访问控制符

外部类的访问控制符只能是 public 和默认（或说缺省，即 default）两种：

// 公共外部类，亦称为“主类”，必须定义在名为 <CLASS_NAME> 的 .java 源文件中。
public class <CLASS_NAME> {}

// 普通的外部类，可以定义在任意的 .java 源文件中。
class <CLASS_NAME> {}

内部类的访问控制符则没有这个限制。

1.3 类与源文件、字节码文件的关系

在一个 .java 源文件中可以定义多个外部类，但只能定义 一个公共外部类（声明为 public class 的类） 。

在编译 .java 源文件时，每个类（不论外部、内部类）都会生成一个对应名称的 .class 字节码文件。外部类生成的字节码文件名为 <CLASS_NAME>.class ，内部类生成的字节码文件名为 <OUTER_CLASS_NAME>$<INNER_CLASS_NAME>.class 。

需要注意的是， 在进行项目开发时，很少会出现在一个 .java 源文件中定义 多个外部类 的情况。

2. main 方法

形如：

public static void main(String[] args) {
    . . .
}

根据 Java 语言规范，main 方法必须声明为 public（不过当 main 方法不是 public 时，有些版本的 Java 解释器也可以执行 Java 应用程序，感兴趣的话，可以在网站 https://bugs.java.com/bugdatabase/index.jsp 上输入 bug 号码 4252539 查看）。

main 方法所在的类称为 主类 ，采用 public class 的方式定义。

每个 Java 应用程序都有一个带 String[] args 参数的 main 方法，如果以如下形式调用某个 Java 程序：

java <PROGRAM_NAME> -g cruel world "HELLO JAVA"

main 方法的 args 参数将会被赋值为：

args[0] -> "-g"
args[1] -> "cruel"
args[2] -> "world"
args[3] -> "HELLO JAVA"

所传的参数称为 命令行参数 。

当没有给 Java 程序传入参数时，Java 程序默认会将 args 赋值为一个长度为 0 的数组 （注意，长度为 0 的数组并不等同于 null）。

3. 注释

  • 单行注释

    // Annotate here

  • 多行注释

    /*
  Annotate here
  */

    注：在 Java 中，多行注释不能嵌套。

  • 文档注释

    /**
  * Annotate here
  */

4. 标识符与关键字

4.1 标识符命名规则、规范

规则：

  • 标识符以 字母 、 下划线（_） 和 美元符号（$） 开头，不能以 数字 开头

    注意，Java 中的“字母”，指在 某种语言中表示字母的任何 Unicode 字符 。如果想要知道哪些 Unicode 字符属于 Java 中的字母 ，可以使用 Character 类的 isJavaIdentifierStart 和 isJavaIdentifierPart 方法来判断。

  • 标识符可以是 任意长度

  • 标识符不可以是 关键字

规范：

  • 标识符的命名应 有意义 ，避免使用缩写

  • 标识符的 命名风格应统一 ，如不应将两个同类的判断方法命名为 CheckIfXxxYyy 和 IsXxxYyy

  • 不要用 美元符号（$） 来命名标识符，习惯上，它只用在机器自动产生的源代码中

  • 变量名、方法名：

    使用首字母小写的驼峰命名法。

  • 类名：

    使用首字母大写的驼峰命名法。

  • 常量名：

    所有字母大写，单词间以下划线连接。

  • 包名：

    一般所有字母小写。

4.2 关键字

true 、 false 、null 三个为特殊的字面值常量，也不可用作标识符。
