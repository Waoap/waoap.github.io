计算机、程序和 Java 概述

2021-12-26 #Computer #Program #Java #Computer Language

本文的内容包括：
1、计算机；
2、编程语言；
3、操作系统；
4、Java 概述。

1.1 计算机

计算机是 存储和处理电子数据的电子设备 ，包括 硬件 和 软件 两部分，硬件包括计算机中可以看得见的物理部分，而软件提供看不见的指令。一台计算机是由以下几个主要的硬件组件构成的：

它们通过一个称为 总线（bus） 的子系统连接。在 PC （个人电脑）中，总线搭建在主板上。

1.1.1 中央处理器

中央处理器（CPU）通常由 控制单元（control unit） 和 算数/逻辑单元（arithmetic/logic unit） 两部分组成。控制单元用于控制和协调其他组件的动作，算术/逻辑单元用于完成数值运算（加法、减法、乘法、除法）和逻辑运算（比较）。

每台计算机都有一个 内部时钟 ，以固定速度发射电子脉冲，用于控制和同步各种操作的步调，其发射频率也即是我们常说的 CPU 频率，单位一般为 兆赫兹（MHz） 和 千兆赫（GHz） 。

最初的 CPU 只有一个 核（core） ，核是 CPU 中实现指令读取和执行的部分。

1.1.2 比特和字节

1 个二进制数为 1 比特（bit） ，8 比特为 1 字节（byte） 。字节是计算机中最小的存储单元。各种类型的数据都根据编码模式（scheme）被编码为字节序列。

1.1.3 内存

内存由一个有序的字节序列组成，每个字节都有唯一的地址，由于可以按任意顺序存取字节，内存被称为 随机访问存储器（Random-Access Memory，RAM） 。内存中字节的内容永远非空但可能对我们的程序来说毫无意义。

1.1.4 存储设备

相较内存慢但不易失的存储器，被称为 只读内存（Read-Only Memory） ，主要有以下三种类型：

  • 磁盘驱动器；
  • 光盘驱动器（CD 和 DVD）；
  • USB 闪存驱动器。

驱动器（drive）是对存储介质进行操作的设备，例如磁盘、光盘。对应介质：

  • 磁盘： 包括硬盘（hard disk）和移动硬盘（mobile hard disk）；
  • 光盘和数字化视频磁盘（CD 和 DVD）： 都分只读和可读写两种（CD-R、CD-RW，DVD-R、DVD-RW）；
  • USB 闪存驱动器： 通用串行总线（Universal Serial Bus，USB）接口允许用户将多种外部设备连接到计算机，包括 USB 闪存驱动器（flash drive），即我们说的 u 盘。

1.1.5 输入输出设备

常用的输入设备包括键盘、鼠标、麦克风、摄像头等；
常用的输出设备包括显示器、音响、打印机等。

1.1.6 通信设备

包括各种调制解调器、DSL、有线网卡接口、无线适配器等。

1.2 编程语言

计算机程序（program）必须使用计算机可以使用的语言编写。编程语言包括如下几种：

1.2.1 机器语言

计算机的原生语言就是 机器语言（machine language） ，即一套内嵌的原子指令集，这些指令都以二进制的形式存在，因而编写程序时必须以二进制代码输入指令。

1.2.2 汇编语言（低级语言）

汇编语言（assembly language） 使用助记符来表示每一条机器语言指令，降低了程序设计的难度，但本质上非常接近机器语言，是机器相关的，使用时需要知道 CPU 是如何工作的，被认为是低级语言。汇编语言使用 汇编器（assembler） 将汇编代码转化为机器代码。

1.2.3 高级语言

高级语言的指令称为语句，其编写的程序称为源程序（source program）或源代码（sourc code）。由于计算机不能运行源程序，必须被翻译成机器代码。其中，翻译由 解释器 或 编译器 来完成。

  • 解释器从源代码中读取一条语句，将其翻译为机器代码或者虚拟机器代码，然后立刻运行，来自源代码的一条语句可能被翻译为多条机器指令。
  • 编译器将整个源代码翻译为机器代码文件，然后执行该机器代码文件。

1.3 操作系统

操作系统（Operating System, OS) 是运行在计算机上的最重要的程序，它可
以管理和控制计算机的活动，其主要任务有：

  • 控制和监视系统活动；
  • 分配和调配系统资源；
  • 调度操作（包括多道程序设计、多线程、多处理等技术）。

1.4 Java 概述

Java最早由 SUN 公司于 1996 年发布，然后来 SUN 公司为 Oracle 公司收购，如今我们使用的便是 Oracle Java。一些资料见 👉Java-Wikipedia 以及 👉Jetbrains 对开发者生态系统的调研报告

Java 是简单的（simple)、面向对象的（object oriented )、分布式的（distributed )、解释型的（interpreted )、健壮的(robust)、安全的（secure)、体系结构中立的（architectureneutral)、可移植的（portable)、高性能的（high performance)、多线程的（multithreaded ) 和动态的（dynamic)。

Java 语言规范（Java language specification) 是对语言的技术定义，包括 Java 程序设计语
言的语法和语义。完整的 Java 语言规范可以在 https://docs.oracle.com/javase/specs/ 上找到。

应用程序接口（Application Program Interface, API）也称为库，包括为开发 Java 程序
而预定义的类和接口。API仍然在扩展，在 Oracle 官网上，可以査看和下载对应版本的 Java API。

Java 分为 JavaSE（Java Standard Edition）、JavaEE（Java Enterprise Edition）、JavaME（Java Micro Edition）三个版本。

  • JavaSE：标准版，定位于个人计算机上的应用（基础）；
  • JavaEE：企业版，定位于服务端的应用（是 SE 在数据库等方面的拓展）；
  • JavaME：微型版，定位于消费性电子产品的应用（基本不接触）。

JDK，即 Java 开发工具包（Java Development Toolkit，JDK）是由一套独立程序构成的集合，用于开发和测试 Java 程序。

Java 程序可以在 Web 浏览器中运行，称为 Java 小程序（applet），内嵌在 HTML 中。我们也可以使用 Java 开发富因特网应用（RIA）。现在，Java 广泛用于开发服务器端的应用程序。

Java 的核心特性与优势是 跨平台、可移植性 （如 Java 的 int 永远是 32 位，不同于 C++ 等可能因编译器不同而产生软件移植时的困难）和 垃圾回收机制 。

1.4.1 跨平台实现：

Java 中： 由 编译器 将 .java 源文件 编译为 .class 字节码文件 ，并由适合各个平台的 JVM 虚拟机 翻译成机器码而后执行。其中，正是由 JVM 虚拟机 充当着翻译官的角色，将字节码文件翻译成各平台对应的可执行文件，以此实现跨平台。（注：在执行 java 程序时，表面上我们使用的是 java.exe 命令来调用 java.exe 程序，但实际上会在此过程中动态调用 JVM，由 JVM 将字节码一行一行地解释成当前平台适合的可执行文件的格式，因此我们认为，Java是一种 “解释型语言” ）

C 语言中： 由 适合各个平台的编译器 将 .c 源文件 编译为各平台对应的可执行文件，以此实现跨平台[1]。

区别： Java 语言的字节码与平台无关，可以使用一份源文件编译出的一份字节码在不同平台上使用；C 语言的编译器与平台相关，编译后的可执行文件也与平台相关，并不属于真正意义上的跨平台，即便是编译前的源文件，也会受到编译器不一致带来的影响。

1.4.2 垃圾回收机制：

垃圾（内存）回收的目的在于主动清除不再使用的对象等，释放内存，减少了因程序设计时没有释放空间而造成内存泄漏的情况的发生。
